## Design Context

### Users
Mixed audience of recruiters/hiring managers and fellow engineers, with engineers as the primary audience. Recruiters scan quickly for credibility signals (role clarity, real projects, professional presentation). Engineers explore projects in depth, read technical blog posts, and evaluate craft quality. Both audiences arrive with intent -- this is not a discovery site but a destination linked from GitHub, resumes, or social profiles.

### Brand Personality
**Precise, confident, understated.**

The site communicates quiet technical authority. No flash, no self-promotion -- just substance presented with care. The voice is calm, concise, and technically confident. It reads as written by a real mobile engineer, not generated by AI or a marketing team.

Emotional goals: visitors should feel they are looking at the work of someone who pays attention to detail, ships real products, and has strong engineering taste.

### Aesthetic Direction
**Minimal editorial developer aesthetic** with subtle engineering/terminal cues.

- Dark mode: Ghostty terminal-inspired palette (deep navy-charcoal background, Catppuccin-influenced text)
- Light mode: macOS editorial aesthetic (soft lavender-tinted white, deep graphite text)
- Single accent family: muted purple (#8b7cf6 dark / #7c6cf5 light) -- used sparingly for links, active states, highlights
- Typography-driven hierarchy: Inter for UI/body, JetBrains Mono for headings, metadata, labels
- macOS window chrome (traffic lights) as a recurring design motif
- Glass/blur effects used with restraint for nav and panels
- No gradients, no neon, no glassmorphism excess, no generic card grids

Anti-references: generic AI portfolios, SaaS landing pages, startup templates, over-decorated personal brands. See `docs/Guidelines.md` for comprehensive anti-AI-slop rules.

### Design Principles

1. **Typography carries the design.** Hierarchy, rhythm, and personality come from type choices, not decoration. Every heading, label, and body block should feel intentional.

2. **Earned presence.** Every element must justify its existence. If it does not communicate information or aid navigation, remove it. Decorative elements are permitted only when they reinforce the technical/engineering identity.

3. **Restrained color, single accent.** The muted purple accent is the only color family beyond neutrals and semantic signals (green/yellow/red). It appears sparingly -- overuse dilutes its impact.

4. **Motion supports clarity.** Animations add tactility and rhythm but never block reading or reduce credibility. All motion respects reduced-motion preferences. Calm and smooth over dramatic.

5. **Authored, not templated.** Layout composition should feel designed by a person with taste -- asymmetric where appropriate, varied across sections, with deliberate whitespace. Avoid patterns that read as default template output.

### Technical Constraints

- **Accessibility**: WCAG AA compliance (contrast ratios, keyboard navigation, screen reader support, focus indicators)
- **Performance**: Static-first with selective hydration (Astro islands). Minimize JS payload.
- **Responsiveness**: Mobile-first considered. Desktop and mobile layouts are both intentionally designed, not collapsed versions of each other.
- **Motion accessibility**: `prefers-reduced-motion` respected system-wide. User toggle available in settings.
- **Theme**: Dark (default) and light modes with smooth transitions. System preference detection supported.

### Design Tokens Reference

- **Fonts**: Inter (300-700), JetBrains Mono (400-500)
- **Base radius**: 10px (with sm/md/lg/xl scale)
- **Container max-width**: 1080px
- **Base font size**: 15-17px (user-configurable: compact/default/large)
- **Accent**: `--accent` CSS variable (#8b7cf6 dark, #7c6cf5 light)
- **Full token system**: `src/styles/theme.css` and `src/styles/tailwind.css`
